Instalacja oprogramowania wymaganego przez AGILE TOOLKIT

Przed rozpoczęciem pracy z Agile Toolkit warto zadbać o poprawne przygotowanie środowiska pracy. Założyliśmy, że do tworzenia projektu będziesz korzystał z lokalnej maszyny, czyli adresem Twojego serwera będzie http://localhost/

Do pracy z Agile Toolkit zalecamy skorzystanie z dowolnej paczki Apache+PHP+MySQL, które są dostępne dla wszystkich systemów operacyjnych. Minimalną wersją PHP jest 5.2, niemniej zalecamy korzystanie z wersji 5.3+.

Podczas pracy z Agile Toolit będzie tworzył dużo plików *.php. Warto uzbroić się w dobry edytor taki jak np. Notepad++ (Windows) lub Coda (Mac OSX)

Powyższe oprogramowanie jest darmowe i można je pobrać pod adresami:

Narzędzia dla Windows

  • WAMP - http://www.wampserver.com/en/
  • Notepad++ - http://notepad-plus-plus.org/

Narzędzia dla Mac OSX

  • MAMP - http://www.mamp.info/en/index.html
  • Coda - http://www.panic.com/coda/

Pobierz i przetestuj

Pobierz Agile Toolkit ze strony agiletoolkit.org i rozpakuj go w głównym katalogu serwera. Jeśli katalog ten używasz w innych celach, możesz rozpakować framework w dowolnym podkatalogu. W pobranym archiwum ZIP znajdziesz Agile Toolkit, dodatki (Add-ons) oraz przykładowy projekt.

Otwórz w przeglądarce plik index.php. Powinieneś zobaczyć "Hello World from your own copy of Agile Toolkit". Możesz porównać wygląd strony z instalacją znajdująca się na naszych serwerach: http://example.agiletoolkit.org/ Jeśli wszystko poprawnie się wyświetliło, możemy przejść dalej.

Alternatywny sposób instalacji

Jeśli korzystasz z Linuxa, prawie na pewno masz zainstalowany serwer Apache. Poza Apachem możesz korzystać z takich serwerów jak Nginx + PHP-FMP lub lighttpd.

Git

Jeśli korzystasz z Gita, zamiast ręcznego ściągania frameworka, możesz sklonować repozytorium atk4-example. Ewentualnie możesz utworzyć własne repozytorium i dodać do niego jako atk4 oraz atk4-addons jako submoduły.

$ git clone git://github.com/atk4/atk4-example $ cd atk4-example $ git submodule init $ git submodule update

Wdrożenie

Jeśli chcesz podzielić się swoją aplikacją z innymi, sugerujemy skorzystać z usługi PaaS (Platform as a Service) oferującej hosting PHP w chmurze. Dzięki użyciu chmury, będziesz mógł skupić się na rozwijaniu aplikacji bez martwienia się o kwestie związane z administrowaniem serwerem. Agile Toolkit został przetestowany pod tym kątem i nie powinien sprawiać żadnych problemów. Zalecamy jedną z dwóch chmur:

  • phpfog — http://phpfog.com/
  • orchestra.io - http://orchestra.io/
